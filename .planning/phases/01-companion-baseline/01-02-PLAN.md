<task type="auto">
  <name>Task 1: Add official Chrome Identity auth configuration and background facade</name>
  <files>
    manifest.json
    lib/export-auth.js
    background.js
  </files>
  <action>
    Implement the export-auth baseline using Chrome MV3 primitives and keep token handling background-owned.

    Required implementation details:
    - Add `identity` permission and an `oauth2` block to `manifest.json`. Use the narrowest baseline scope from research: `https://www.googleapis.com/auth/drive.file`.
    - Preserve existing permissions and host permissions unless a Phase 1 need requires a targeted cleanup. Do not add broader Drive scopes.
    - Create `lib/export-auth.js` and load it from `background.js` via `importScripts(...)`.
    - In `lib/export-auth.js`, implement:
      - `beginExportAuth()` calling `chrome.identity.getAuthToken({ interactive: true, scopes: ['https://www.googleapis.com/auth/drive.file'] })`
      - `getExportAuthStatus()` returning sanitized status only (`authorized`, `grantedScopes`, `accountHint`, `lastError`, `updatedAt`)
      - `clearExportAuth()` using the Chrome Identity token-removal path and resetting sanitized status
    - Do not persist raw tokens, refresh tokens, client secrets, or full OAuth responses in any extension storage area.
    - Keep this auth domain separate from `NotebookLMAPI.getTokens()` and account scraping. Phase 1 must not conflate NotebookLM import auth with Google export auth.
  </action>
  <verify>
    <automated>node -e "const fs=require('fs'); const manifest=JSON.parse(fs.readFileSync('manifest.json','utf8')); if(!(manifest.permissions||[]).includes('identity')) throw new Error('identity permission missing'); if(!manifest.oauth2) throw new Error('oauth2 block missing'); if(!(manifest.oauth2.scopes||[]).includes('https://www.googleapis.com/auth/drive.file')) throw new Error('drive.file scope missing'); const auth=fs.readFileSync('lib/export-auth.js','utf8'); if(!/chrome\\.identity\\.getAuthToken/.test(auth)) throw new Error('getAuthToken missing'); if(/accessToken|refreshToken/.test(auth) && /storage\\.(sync|local)\\.set/.test(auth)) throw new Error('raw token persistence risk'); console.log('auth baseline ok');" && node --check background.js && node --check lib/export-auth.js</automated>
  </verify>
  <done>
    The manifest declares official extension auth with `drive.file`, the background owns token acquisition, and no raw Google credentials are persisted to extension storage.
  </done>
</task>
